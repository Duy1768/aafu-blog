Kiểu dữ liệu nguyên thủy và kiểu dữ liệu tham chiếu

Trong JavaScript, biến lưu trữ hai loại dữ liệu cơ bản: kiểu nguyên thủy và kiểu tham chiếu. Hiểu rõ sự khác biệt giữa hai loại này là điều cần thiết để quản lý bộ nhớ và kiểm soát việc chia sẻ, lưu trữ và thay đổi dữ liệu.

1. Kiểu nguyên thủy (Primitive Types)

Kiểu dữ liệu đơn giản nhất được gọi là kiểu nguyên thủy. Chúng lưu trữ trực tiếp dữ liệu không thay đổi trong biến. Các kiểu nguyên thủy mà JavaScript hỗ trợ:

  • string: "hello"
  • number: 42
  • boolean: true hoặc false
  • null
  • undefined
  • symbol
  • bigint

Đặc điểm chính:

  • Bất biến: Giá trị không thể thay đổi trực tiếp.
  • Lưu trữ theo giá trị (value).

2. Kiểu tham chiếu (Reference Types)

Kiểu tham chiếu lưu trữ vị trí bộ nhớ của đối tượng, thay vì lưu giá trị trực tiếp. Các biến lưu một tham chiếu đến địa chỉ bộ nhớ.

Các ví dụ:

  • object: { name: 'Alice' }
  • array: [1, 2, 3]
  • function: function() { console.log('hello'); }
  • Date: new Date()

Đặc điểm chính:

  • Có thể thay đổi (mutable): nội dung có thể sửa đổi.
  • Lưu trữ theo tham chiếu (reference).

3. Ví dụ minh họa

// Primitive Example
let a = 10;
let b = a;
b = 20;
console.log(a); // Output: 10

// Reference Example
let obj1 = { name: 'Alice' };
let obj2 = obj1;
obj2.name = 'Bob';
console.log(obj1.name); // Output: 'Bob'

Giải thích:

Kiểu nguyên thủy: Sao chép giá trị độc lập, thay đổi b không ảnh hưởng a.

Kiểu tham chiếu: Cả obj1 và obj2 trỏ đến cùng một đối tượng, thay đổi obj2 cũng ảnh hưởng obj1.

4. Thay đổi (Mutation) vs Gán (Assignment)

Thay đổi (Mutation): sửa đổi nội dung đối tượng hiện có.

let arr = [1, 2, 3];
let arr2 = arr;
arr2.push(4);
console.log(arr); // [1, 2, 3, 4]

Gán (Assignment): thay đổi tham chiếu sang đối tượng mới.

let arr = [1, 2, 3];
let arr2 = arr;
arr2 = [4, 5, 6];
console.log(arr); // [1, 2, 3]

5. Sao chép đối tượng và mảng

Sao chép nông (Shallow Copy)

let original = { name: 'Alice' };
let copy = { ...original };
copy.name = 'Bob';
console.log(original.name); // 'Alice'

Sao chép sâu (Deep Copy)

let nested = { person: { name: 'Alice' } };
let deepCopy = JSON.parse(JSON.stringify(nested));
deepCopy.person.name = 'Bob';
console.log(nested.person.name); // 'Alice'

6. Truyền theo giá trị vs truyền theo tham chiếu

Kiểu nguyên thủy (truyền theo giá trị):

function modifyValue(x) {
  x = 20;
}
let num = 10;
modifyValue(num);
console.log(num); // 10

Kiểu tham chiếu (truyền theo tham chiếu):

function modifyObject(obj) {
  obj.name = 'Bob';
}
let person = { name: 'Alice' };
modifyObject(person);
console.log(person.name); // 'Bob'

7. Kiểu bao đóng nguyên thủy (Primitive Wrapper Types)

Mặc dù kiểu nguyên thủy là bất biến, JavaScript tạm thời bao đóng chúng trong đối tượng để truy cập phương thức/thuộc tính:

let str = 'hello';
console.log(str.length); // 5

8. Best practices

Sử dụng const cho kiểu tham chiếu: ngăn gán lại nhưng vẫn cho phép thay đổi nội dung.

const obj = { name: 'Alice' };
obj.name = 'Bob'; // Allowed
obj = { age: 25 }; // Error

Tránh thay đổi không mong muốn: sử dụng spread operator hoặc sao chép sâu.

Biết khi nào sử dụng sao chép sâu: cấu trúc lồng nhau cần deep copy.

Tận dụng tính bất biến: áp dụng kỹ thuật lập trình hàm hoặc thư viện như Immutable.js.

9. Những cạm bẫy thường gặp

Nhầm lẫn giữa gán và thay đổi.

Sửa đổi tham chiếu chung có thể gây ra hậu quả không mong muốn.

Sao chép nông không bảo vệ các thay đổi trong cấu trúc lồng nhau.
